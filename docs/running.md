# Running Aratiri

This guide covers local and container startup. For environment variables and secrets, see [Configuration Reference](configuration.md).

## Requirements

- Java 25. The Gradle toolchain is configured to auto-download when possible.
- Docker, if using Compose or Testcontainers-backed integration tests.
- PostgreSQL.
- Kafka.
- An LND node reachable over gRPC.
- LND admin macaroon and TLS material.

## Secrets

Aratiri authenticates to LND with the configured admin macaroon path and optional TLS certificate path.

For a typical local LND install, export the admin macaroon as hex:

```bash
xxd -p ~/.lnd/data/chain/bitcoin/mainnet/admin.macaroon | tr -d '\n' > secrets/admin.macaroon
```

If you need a custom certificate, place it somewhere readable by the app, for example:

```text
secrets/tls.cert
```

Then point `ADMIN_MACAROON_PATH` and `LND_TLS_CERT_PATH` at those files.

## Local Gradle Run

Use Gradle outputs from `build/`. Do not run old `target/` or IDE output artifacts because gRPC/protobuf classes are generated by the Gradle build.

```bash
cp .env.example .env
```

Edit `.env`, then either export it into your shell or configure the variables in your IDE.

Build and run:

```bash
./gradlew clean build
./gradlew bootRun
```

Or run the packaged boot jar:

```bash
./gradlew clean bootJar
java -jar build/libs/aratiri-*.jar
```

When running locally against Compose services rather than from inside the Compose network, use a localhost Kafka bootstrap server, for example:

```text
KAFKA_BOOTSTRAP_SERVERS=localhost:9092
SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/aratiri_db
```

## Docker Compose

Compose defines separate profiles for PostgreSQL, Kafka, the backend, and the full stack.

Start PostgreSQL only:

```bash
docker compose --profile db-only up -d
```

Start Kafka only:

```bash
docker compose --profile kafka-only up -d
```

Build and start the backend with its dependencies:

```bash
docker compose --profile full-stack up --build -d
```

The backend listens on `http://localhost:2100` by default. The Compose file also includes `aratiri-admin` on port `3001` and `aratiri-frontend` on port `3000`.

## Useful URLs

- API base URL: `http://localhost:2100`
- Swagger UI: `http://localhost:2100/swagger-ui.html` when `ARATIRI_SECURITY_API_DOCS_ENABLED=true`
- OpenAPI JSON: `http://localhost:2100/v3/api-docs` when `ARATIRI_SECURITY_API_DOCS_ENABLED=true`
- H2 console: permitted only in `dev`, `development`, or `test` profiles unless `aratiri.security.dev-endpoints.h2-console-enabled=true`
- LNURL metadata example: `http://localhost:2100/.well-known/lnurlp/{alias}`
- WebSocket notifications: `ws://localhost:2100/v1/notifications/subscribe?token={jwt}`

## Tests And Verification

Run the repository-required verification command after code changes:

```bash
./gradlew clean build
```

The build runs unit and integration tests, SonarLint checks, SpotBugs/FindSecBugs analysis, JaCoCo coverage verification, and `verifyProtoClassesInBootJar`. Integration tests use Testcontainers for infrastructure where needed.

## Postman

The repository includes a local Postman collection:

- [Postman Collection Guide](../postman/README.md)
- `postman/Aratiri.local.postman_collection.json`
- `postman/Aratiri.local.postman_environment.json`

The collection assumes the local API is available at `http://localhost:2100` and includes a demo flow using seeded test users.
